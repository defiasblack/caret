# Caret Roadmap

This roadmap describes the project's current direction. It is not a promise of specific features or delivery dates, and priorities may change based on user feedback, maintenance needs, and contributor availability.

## Near-term priorities

- Improve reliability across Windows, Linux, macOS, and SSH sessions
- Expand automated tests for editing, navigation, tabs, project-tree behavior, and terminal integration
- Make installation and upgrades easier with documented release artifacts
- Improve error messages and recovery when files, shells, language servers, or plugins fail
- Continue accessibility work for keyboard-only, monochrome, and reduced-motion use
- Keep documentation, built-in help, and command discovery synchronized

## Editor experience

- Refine selections, multi-occurrence editing, undo grouping, and clipboard behavior
- Improve large-file responsiveness and rendering efficiency
- Expand configurable keymaps without sacrificing approachable defaults
- Strengthen session and workspace restoration

## Coding intelligence

- Broaden and harden language-server support
- Improve completion, diagnostics, references, rename, formatting, and code-action workflows
- Add clearer language-server setup and troubleshooting guidance
- Expand syntax highlighting while keeping startup and editing responsive

## Project and terminal workflows

- Improve project-tree operations and file-management commands
- Refine integrated terminal resizing, scrolling, shell detection, and cross-platform behavior
- Improve Git awareness and project navigation

## Plugins and customization

- Stabilize and document the plugin manifest and lifecycle
- Add more examples for commands, language rules, themes, and save hooks
- Define compatibility expectations as the plugin API evolves
- Improve theme and settings discoverability

## Project health

- Maintain continuous integration on all supported desktop platforms
- Automate dependency updates and release checks
- Publish clear changelogs and migration notes
- Grow contribution documentation and newcomer-friendly issues

## Suggesting priorities

Use the feature-request template to propose an addition or improvement. Describe the user problem, expected workflow, and why the request fits Caret's goal of being a polished, approachable terminal editor.
